The SLA Management System has been deployed successfully!

1. Check the deployment status:

   kubectl get pods -n {{ .Release.Namespace }} -l app.kubernetes.io/name={{ include "sla-management-system.name" . }}

2. Access the API server:

   kubectl port-forward -n {{ .Release.Namespace }} svc/{{ include "sla-management-system.fullname" . }} 8000:8000

   Then access the API at http://localhost:8000

3. API Endpoints:

   - Health Check: GET /health
   - Ready Check: GET /ready
   - SLO Management: /api/v1/slos
   - Budget Management: /api/v1/budgets
   - Policy Management: /api/v1/policies
   - Webhook Receiver: POST /webhooks/alertmanager
   - API Documentation: /docs

4. Metrics:

   kubectl port-forward -n {{ .Release.Namespace }} svc/{{ include "sla-management-system.fullname" . }} 9090:9090

   Then access metrics at http://localhost:9090/metrics

5. View logs:

   kubectl logs -n {{ .Release.Namespace }} -l app.kubernetes.io/name={{ include "sla-management-system.name" . }} -f

6. CRD Management:

   # List SLO Definitions
   kubectl get slodefinitions --all-namespaces

   # List SLA Policies
   kubectl get slapolicies --all-namespaces

   # Create a new SLO
   kubectl apply -f examples/sla-management-system/example-slo-latency.yaml

   # Create a new Policy
   kubectl apply -f examples/sla-management-system/example-policy-orchestration-freeze.yaml
